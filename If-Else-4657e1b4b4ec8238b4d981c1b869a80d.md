
Python avalia condições usando operadores como **`==`****,** **`!=`****,** **`<`****,** **`<=`****,** **`>`****,** **`>=`**.  O principal uso dessas condições é em **declarações** **`if`**, **que executam código quando a condição for verdadeira**.

    > **Indentação** é obrigatória para definir blocos de código após o `if` — não existem chaves como em outras linguagens.

Um simples comando `if` verifica uma condição e executa todo o bloco se o resultado for `True`:


    ```python
    if b > a:
        print("b é maior que a")
    ```

    - **Elif** verifica outra condição se a anterior for falsa.
    - **Else** executa quando nenhuma das condições anteriores for verdadeira.
    - É possível combinar múltiplas condições com **`and`** / **`or`**.
    - **Nested if** são `if` dentro de outro `if`.
    - **Pass** é usado quando um bloco `if` é necessário, mas você não quer que nada aconteça ainda.

###  **If Elif**

    - `elif` significa “else if”: verifica uma nova condição **se a condição anterior foi falsa**.
    - Você pode ter quantos `elif` quiser em uma cadeia de decisão.
    - Apenas o primeiro bloco com condição verdadeira é executado; os demais são ignorados.

###  **If … Else**

    - O bloco `else` **captura tudo que não foi tratado** por `if` ou `elif`.
    - É útil para definir um comportamento padrão quando nenhuma condição é satisfeita.

        ```python
        if b > a:
            …
        elif a == b:
            …
        else:
            print("a é maior que b")
        ```


###  **If Shorthand**

    - Python permite escrever condicionais **em uma linha** quando o bloco tem apenas um comando.
    - **Shorthand if**:

        ```python
        if a > b: print("a é maior que b")
        ```

    - **Shorthand if-else (ternário)**:

        ```python
        print("A") if a > b else print("B")
        ```


###  **If Logical (Operadores Lógicos)**

    - Condições podem ser combinadas com operadores lógicos:
        - **and** → todas as condições precisam ser verdadeiras.
        - **or** → ao menos uma condição verdadeira.
        - **not** → inverte o valor lógico.
    - Exemplo:

        ```python
        if a > b and c > a:
            print("Ambas são verdadeiras")
        ```

    - Isso permite testar cenários mais complexos de decisão. ([w3schools.am](https://www.w3schools.am/python/python_conditions.html?utm_source=chatgpt.com))

###  **If  dentro de outro If**

    - **Nested if** são simplesmente declarações `if` dentro de outra.
    - Permitem decisões **em múltiplos níveis**:

        ```python
        if x > 10:
            print("Acima de 10")
            if x > 20:
                print("E também acima de 20")
            else:
                …
        ```

    - Útil quando uma condição só faz sentido dentro de outra.

### **If Pass**

    - Em Python, um bloco `if` **não pode ficar vazio** — isso causa erro de sintaxe.
    - Para evitar isso sem executar código, usamos **`pass`**:

        ```python
        if b > a:
            pass
        ```

    - `pass` indica que nada deve ser feito, mas o bloco ainda é sintaticamente válido. ([w3schools.am](https://www.w3schools.am/python/python_conditions.html?utm_source=chatgpt.com))

## 📌 **Resumo geral (visão conceitual)**


| Conceito                     | O que faz                                             | Quando usar                               |
| ---------------------------- | ----------------------------------------------------- | ----------------------------------------- |
| `if`                         | Testa uma condição simples                            | Sempre que precisar de lógica condicional |
| `elif`                       | Testa outra condição se a anterior não for verdadeira | Para múltiplas decisões sequenciais       |
| `else`                       | Bloco padrão quando nenhum `if/elif` é verdadeiro     | Para casos “coringa”                      |
| Shorthand one-line           | Versão curta do `if` / `if-else`                      | Quando bloco é curto                      |
| Lógicos (`and`, `or`, `not`) | Combinam múltiplas condições                          | Para cenários compostos                   |
| if aninhados                 | `if` dentro de `if`                                   | Para decisões hierárquicas                |
| `pass`                       | Bloco vazio válido                                    | Quando ainda não implementou código       |
